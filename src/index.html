Animation Assistant

Введение

Плагин animation-assistant помогает работать с анимацией. Плагин работает со следующими CSS библиотеками:

  • animate.css 4.0.0^
  • cssanimation 1.0.0^
  • hover.css 2.0.0^
  • CSShake 1.5.0^
  • magic.css 1.1.0^
  • tuesday 1.2.0^
  • repaintless 1.3.0^
  • animista
  • Возможно, ваша любимая библиотека CSS анимации
  • Работа со своей анимацией через плагин

Начало работы

Установите плагин в ваш проект используя NPM 

            
    npm install --save animation-assistant

Подключите его удобным вам способом:

CommonJS: 

            
    const AnimationAssistant = require('AnimationAssistant');

ESM: 

            
    import AnimationAssistant from 'AnimationAssistant';

Для начала необходимо получить экземпляр класса AnimationAssistant вызвав соответствующий конструктор и передав ему CSS селектор. Например:

HTML

            
    <div class="forAnimation">  </div>

JavaScript

            
    const animation = new AnimationAssistant(document.querySelectorAll('selector'));

Это может быть как один элемент так и группа элементов:

HTML

            
    <div class="your-class">    </div>
    <div class="your-class">    </div>
    <div class="your-class">    </div>

JavaScript

            
    const animation = new AnimationAssistant(document.querySelectorAll('your-class'));

Интерфейс:

animation.addClasses(string[]): AnimationAssistant - Принимает массив с классами и добавляет их к элементам. Возвращает текущий экземпляр класса.

animation.hiddenLaunch(): AnimationAssistant - Скрывает элемент до начала анимации. Возвращает текущий экземпляр класса.

animation.adaptation(): AnimationAssistant - Адаптирует окружение элемента для корректной работы анимаций. Возвращает текущий экземпляр класса.

cancelOnStartScreen(): AnimationAssistant - Не анимировать элемент если при загрузке страницы он оказался на стартовом экране. Возвращает текущий экземпляр класса.

animation.sequence(interval: number = 200): AnimationAssistant - вызов этого метода будет означать начало работы не с целевым элементом, а с его потомками. Каждому элементу будет задано свойство animation-delay с промежутками указанном в параметре interval. Полезен для анимаций работающих с внутренним контентом содержимого. Например - буквами текста в библиотеке animation.io. Возвращает текущий экземпляр класса.

animation.setLibrary(library: string): animator - Принимает один аргумент в виде строки. Возвращает экземпляр класса animator с методами для настройки анимации. Подготавливает выбранную CSS библиотеку для работы. Основной метод, который устанавливает выбранную вами библиотеку для работы с анимацией. Возвращает экземпляр класса Animator обладающего такими методами как setAnimation и play.

animation.setAnimation(offset: number, name: string, animationEnd: function): Playback - offset - это значение от 1 до 100 (%) относительно верхнего края элемента до верхнего края окна пользователя. Параметр name - это текущий класс, который будет задействован по сигналу от параметра offset. Параметр animationEnd запустит функцию после окончания анимации. Настраивает анимацию для работы. Возвращает текущий экземпляр класса Animator. Функция обратного вызова animationEnd может вернуть другую функцию обратного вызова со следующей сигнатурой: animationEndCallback(target: HTMLElement): void, где target - это текущий элемент не зависимо от контекста вызова. Возвращает экземпляр класса Playback.

animation.play(name: string, timeout: number = 2000): Promise - name - это текущий класс, который будет применен к элементу, timeout задает время до запуска следующей анимации. Запускает анимацию. Возвращает Promise. Возвращает экземпляр класса Animator.

Информация о работе плагина

Методы addClasses, hiddenLaunch, adaptation, sequence и cancelOnStartScreen возвращают экземпляр класса AnimationAssistant. Метод setLibrary возвратит экземпляр класса Animator, который владеет методом для настройки анимации - setAnimation. Данный метод возвращает экземпляр класса Playback с методом play для проигрывания анимации.

Метод play является асинхронным и возвращает Promise. Используйте его для анимаций элементов на стартовом экране или в собственных событиях для привлечения внимания пользователей к элементу. При этом в обработчике событий рекомендуется использовать поле once: true.

Метод setAnimation настраивает параметр offset только для элементов, которые будут встречаться при прокрутке вниз. Если прокручивать страницу вверх данный параметр будет проигнорирован а элемент будет использовать анимацию сразу как только его нижний край сравняется с верхним краем окна браузера.

Советы

Не используйте анимацию на корневом элементе или на BODY. Это возможно, но делать этого не следует.

Не используйте hiddenLaunch, если не собираетесь использовать метод setAnimation. Не используйте hiddenLaunch если основной метод при работе с анимацией у вас это addClasses.

Не используйте анимацию просто так. Анимация - это элемент взаимодействия с пользователем на сайте. Анимацией можно привлечь внимание, напомнить что либо или дать понять что действие было выполнено.

Используйте одну библиотеку. Плагин поддерживает разные библиотеки, но не стоит использовать их все разом. Не желательно использовать более одной библиотеки на сайте.

Метод setAnimation назначает обработчик события scroll на каждый элемент с которым работает. Если вы имеете на сайте 10 экземпляров объекта AnimationAssistant и в каждом из них по 10 элементов, то у вас в наличии будет 100 обработчиков на объекте window. Плагин будет удалять завершенные обработчики с документа в целях производительности.

Чаще всего вы захотите использовать параметр offset в значении 100. В этой позиции элемент будет использовать анимацию сразу после того, как его верхний край сравняется с нижним краем окна браузера. Сначала может показаться, что анимация может быть не видна в некоторых ситуациях, но большинстве случаев это именно то что нам нужно, так как пользователи могут прокручивать страницу очень быстро и просто не заметить анимации, если этот параметр будет установлен в более низком значении, например 10 или 20.

Библиотека может предоставить анимацию, которая будет в бесконечном режиме. В этом случае вы должны быть готовы к ряду ограничений среди которых некорректная работа метода play и невозможность вызвать callback функцию в методе setAnimation. Если бесконечная анимация произошла в методе setAnimation, то метод play не будет выполнен.

Вы можете управлять количеством повторений анимации через метод addClasses. Просто создайте следующий CSS класс: 

            
    .iteration-count {
        animation-iteration-count: %any number% !important;
    }

Затем можно добавить его через метод addClasses и изменить количество повторений у анимации. Это может быть очень полезно, если в библиотеке имеется модификатор infinity там, где он не нужен. Вы сможете управлять количеством повторений анимации для своих нужд или просто удалить им бесконечный режим проигрывания.

Если вы не хотите чтобы анимация проигрывалась на планшетных устройствах или мобильных телефонах, тогда добавьте в CSS класс, который вы использовали при выборке элементов плагином и задайте ему два поля на интересующей вас точке останова - visibility: visible !important и animation-name: none !important. Например, у вас есть селектор классов for-animation и вы отталкиваетесь от этого при использовании плагина: 

            
    new AnimationAssistant('for-animation')
        .setLibrary('library')
        .setAnimation(...settings);

Вы не хотите чтобы эта анимация проигрывалась на устройствах размеры экрана которых меньше 670px. Просто добавьте следующие правила в css: 

            
    @media screen and (max-width: 670px) {
        .for-animation {
            visibility: visible !important;
            animation-name: none !important;
        }
    }

                .then((res) => res('animate__wobble'));

        { once: true },

Иногда понадобиться дополнительные классы. Например, для библиотеки animate.css это могут быть animate__slower для более медленного проигрывания анимации. Добавлением классов занимается метод addClasses. 

            
    const animation = new AnimationAssistant('animation-css-example-add-classes')
        .addClasses(['animate__slower', 'animate__repeat-2'])
        .adaptation()
        .hideAtStart()
        .setAnimation(20, 'animate__backInRight');

Так-же не забывайте про callback функцию, которая выполнится после завершения анимации. Вы можете передать ее третьим параметром: 

            
    const animation = new AnimationAssistant('animation-css');
    .addClasses(['animate__slow'])
    .adaptation()
    .hideAtStart()
    .setAnimation(30, 'animate__bounceInLeft', () => {
        // ваш код, который будет выполнен после завершения анимации animate__bounceInLe
        // выполнится единожды, даже если задано повторение анимации
        // например при помощи класса animate__repeat-2 библиотеки animate.css
        // помните, что если анимация находится в бесконечном режиме (infinity),
        // то этот код не выполнится никогда
        return () => {
            // этот код будет выполнен самым последним после выполнения всего кода
            // включая метод play и все его then
        };
    });

При работе с этой библиотекой следует обратить внимание на две важные особенности. Во-первых, многие ее анимации имеют свойство animation-iteration-count со значением infinite, а во вторых некоторые её анимации работают с элементами span, которые вложены в контейнер. В самой документации они обозначаются с классом sequence, который написан там только для удобства, что-бы понять когда анимация будет работать не с самим элементом, а с его потомками. Метод play не будет работать, потому что предыдущая анимация никогда не закончится, следовательно, новая не начнется.

Например, в следующем примере callback функция не выполнится никогда, так как анимация длится вечно: 

            
    new AnimationAssistant('cssanimation-example-infinity')
        .setLibrary('cssanimation.io')
        .setAnimation(20, 'heartbeatFast', () => {
        // эта функция не будет запущена
    });

Можно использовать метод play для последовательного воспроизведения ряда анимации. Каждый элемент будет придерживаться своему состоянию, а анимация остановится на последней инструкции и будет проигрывать ее при наведении. Необходимо использовать метод setAnimation в значении 100. 

            
    new AnimationAssistant(document.querySelectorAll('.hover-css-to-left'))
        .setLibrary('hover.css')
        .setAnimation(100, 'hvr-wobble-to-top-right', (event)  => {
            // функция обратного вызова будет выполнена после проигрывания установленной анимации
            return () => {
                // эта функция не будет выполнена!
            }
        })
        .play('hvr-buzz-out', 1000)
        .then((res) => res('hvr-bounce-in'));

Работа с другими CSS библиотеками для анимаций, которые не вошли в список

Вы могли не увидеть вышу любимую библиотеку CSS анимации в списке выше. Это не означает, что данный плагин с нею не работает. Возможно они действительно могут быть не совместимы, но скорее всего это не так.

Все, что вам для этого нужно, это метод экземпляра объекта AnimationAssistant - addClasses и два метода класса Animator - play и setAnimation.

Для начала следует установить вашу библиотеку в проект. Затем нужно выбрать элементы и присвоить им CSS класс. 

            
    new AnimationAssistant(document.querySelectorAll('.your-selector'));

В библиотеке вам нужно найти основной класс для работы с анимацией. Обычно он содержит два поля - animation-duration и animation-fill-mode. Например, для библиотеки tuesday это animated, а для magic.css это magictime. Вызовите метод addClasses и добавьте туда этот класс. 

            
    new AnimationAssistant('your-selector')
        .addClasses(['your-class'])ж

Потом обязательно вызовите метод setLibrary передав в него пустую строку, так как вам нужно получить экземпляр класса Animator для доступа к методам play и setAnimation.

Далее следуйте документации библиотеки и выбирая интересующие вас классы добавляйте их в метод play или setAnimation.

Работа со своей анимацией через плагин

Вы можете использовать плагин для работы со своими анимациями. Для этого создайте экземпляр класса AnimationAssistant и передайте ему CSS селектор элементов, которые вы планируете анимировать. Затем вызовите у вновь созданного объекта метод addClasses и передайте туда ваш основной CSS класс при работе с анимациями. Он должен содержать как минимум одно поле - animation-duration.
